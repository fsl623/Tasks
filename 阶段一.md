# Github问答

- ### 据我在网上的了解Github是一个开源分享代码的网站

- ### Pull request是拉取请求，它建立在fork之上，如果我们Fork了一个他人的项目，对其进行了修改，而且感觉修改的还不错，我们就可以对原项目的拥有者提出一个请求，等其对我们的请求审核，并且通过审核之后，就可以把我们修改过的内容合并到原项目之中，这时我们就成了该项目的贡献者
  
- ### fork是分支，如果我们对某一个项目比较感兴趣，并且想在此基础之上开发新的功能，这时我们就可以Fork这个项目，这表示复制一个完成相同的项目到我们的 GitHub 账号之中，而且独立于原项目
 
- ### Pull request与fork是在搜索github学习教程时学习到的
