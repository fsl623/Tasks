# 初识GitHub

#### GitHub 是一个面向开源及私有软件项目的托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub

# GitHub术语解释

- #### Repository：可以理解为是仓库，项目存放在仓库里。如果要创建项目就需要创建仓库来存放项目

- #### Issues：就是问题，如果别人发现了我们项目的bug，就可以提出issues，当然我们也可以给他人提出问题

- #### Star：是点赞相当于抖音中的点赞或收藏，我们点赞过的项目都会存放在Star中

- #### fork：是拉分支，当我们想在某一个项目上发展新的功能，我们就可以复制这个项目到自己的账号中进行创新

- #### Pull Request：是提交请求，此功能建立在fork上，当我们fork别人的项目并修改，想将修改的代码并入到原作者的项目中，就可以用此功能，当然原作者需要审核我们的请求，审核通过后，就可以把改过的代码并入到原项目中，就成为了这个项目的贡献者

- #### Merge：可以理解为合并，就相当于是Pull Request作者方向，我们成为了同意请求的一方

- #### Watch：英译中是“看”，引申一下就是“观察”，当我们观察了一个项目后，这个项目有了任何更新，我们就会第一时间收到更新的消息

- #### Gist：如果只想简单分享一段代码的话就可以选Gist
